## Mission

Radium Desk exists to help every support agent deliver fast, consistent, accountable, and delightful customer service.

The software must reduce customer waiting time, improve internal collaboration, and create complete operational transparency.

---

## Product Vision

Radium Desk is not a traditional Service Desk.

**Radium Desk is an Operations Workspace.**

Every customer interaction should begin and end from a single **Order Workspace**.

The dashboard is only an inbox.

The Order Workspace is where work happens.

---

## Product Principles

### 1. One Workspace

Everything related to one customer should be accessible from one page.

Avoid forcing agents to switch between multiple modules.

### 2. One Timeline

Every important activity should automatically create a timeline entry.

Examples:

- Payment received
- WhatsApp sent
- Email delivered
- Engineer assigned
- Refund approved
- Pickup completed

Timeline should become the single source of operational history.

### 3. One Click

Frequently used actions should require one click.

Examples:

- Call
- WhatsApp
- Email
- Add Note
- Assign Owner
- Schedule Pickup

### 4. One Truth

Each type of information has exactly one source of truth.

| Domain | Source of truth |
|--------|-----------------|
| Payments | Cashfree |
| Orders, Products, Warranty | RadiumBox |
| Service Workflow, Timeline, Communication, Ownership | Radium Desk |

Never duplicate ownership of business data.

### 5. One Goal

Every feature should reduce effort for support agents.

If a feature increases complexity without improving customer service, reconsider it.

---

## UX Principles

The interface must be designed for non-technical support agents.

The system should require minimal training.

**Use:**

- Clear labels
- Readable typography
- Large action buttons
- Visual status chips
- Progressive disclosure

**Avoid** exposing technical IDs unless necessary.

---

## Order Workspace

The Order Workspace becomes the permanent center of Radium Desk.

It should eventually include:

- Overview
- Timeline
- Payments
- Device
- Communication
- Files
- Notes

Future integrations (Cashfree, RadiumBox, WhatsApp, Email, AI) should enhance this workspace instead of creating separate modules.

---

## Timeline Philosophy

Timeline is not a log.

Timeline tells the complete story of the customer journey.

Every entry should answer:

| Question | Meaning |
|----------|---------|
| **Who?** | Actor (agent, system, integration) |
| **What?** | Action or event |
| **When?** | Timestamp |
| **Why?** | Reason or context (where applicable) |

---

## Communication Philosophy

Communication is a core workflow.

Support:

- Phone
- WhatsApp
- Email

Every interaction should be logged automatically.

Future AI assistance should recommend replies but never replace human approval.

---

## Accountability

Every important action should record:

- **Actor** — who performed the action
- **Timestamp** — when it occurred
- **Reason** — where applicable

The system should eliminate ambiguity.

---

## Recognition

The product should eventually support positive performance indicators, including:

- SLA achieved
- First-contact resolution
- Fast response
- Positive customer feedback

Performance is not only about finding mistakes.

---

## Compliance

Compliance should happen automatically.

Audit trails should be generated by normal work instead of requiring extra manual effort.

---

## Product Decision Filter

Before implementing any feature, answer these questions:

1. Does it improve customer service?
2. Does it reduce handling time?
3. Does it reduce clicks?
4. Does it improve collaboration?
5. Does it strengthen accountability?
6. Does it improve compliance?
7. Does it belong inside the Order Workspace?

If the answer to most questions is **No**, reconsider the feature.

---

## Success Metrics

Track measurable outcomes such as:

| Metric | Description |
|--------|-------------|
| Average Time to Answer (ATTA) | Time from customer contact to first agent response |
| First Contact Resolution | Issues resolved without follow-up |
| SLA Compliance | Adherence to service level agreements |
| Customer Response Time | Time for customers to receive replies |
| Timeline Completeness | Coverage of operational events in timeline |
| Agent Productivity | Work completed per agent over time |
| Customer Satisfaction | Direct feedback and satisfaction signals |
| Internal Escalations | Volume and rate of escalations |
| Communication Response Time | Latency across phone, WhatsApp, and email |

Future roadmap decisions should be evaluated against these metrics.
